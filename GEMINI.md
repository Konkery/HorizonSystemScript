# Project Manifest: Horizon Automation System Scripts (27/2025-N1-МАС)

## 1. Mission & Philosophy
Этот репозиторий содержит системные скрипты и модули для развертывания промышленной платформы **Horizon Automation** на аппаратных хабах (Raspberry Pi CM4/CM5).
- **Принципы:** Инфраструктура как код (IaC), Идемпотентность, Безопасность (динамическая генерация паролей).
- **Стек:** Node.js, Node-Red, MongoDB, Redis, Graylog, Grafana.

## 2. Project Hierarchy
- `Common/`: Ядро, загрузчик конфигурации (`config-loader.js`), общие утилиты.
- `LinuxScript/`: Настройка ОС, генерация `systemd` сервисов.
- `MongoDBscript/`: Инициализация СУБД, управление БД/пользователями/коллекциями.
- `NodeRedScript/`: Настройка среды Node-Red, аутентификация, синхронизация.

## 3. Metadata & Versioning Protocol
Все исполняемые файлы (`js`) и документы (`md`) должны содержать блок метаданных в начале файла:
```html
<div align="left">
  <b>ООО "МАС"</b><br />
  <b>Проект:</b> 27/2025-N1-МАС<br />
  <b>Ревизия:</b> rev.XX<br />
  <b>Версия:</b> vXX<br />
  <b>Дата:</b> ДД месяц ГГГГ г.
</div>
```
- **Имена файлов:** Не должны содержать индексов ревизий/версий.
- **Инкремент:** При любом изменении — `vXX+1`. При значительных изменениях логики — `rev.XX+1`, `v01`.

## 4. Technical Guardrails
- **MongoDB:**
  - Использовать `authSource=admin` во всех строках подключения.
  - Идемпотентность: Скрипты должны сначала проверять состояние (наличие БД/пользователей), затем применять изменения.
  - Очистка: Модуль `03__MongoDB-CreateUserAndCollection.js` принудительно удаляет все БД, отсутствующие в `settings.json` (кроме системных).
- **Credentials:**
  - Пароли генерируются динамически через `config-loader.js`.
  - Отчеты сохраняются в `res/01__config/CREDENTIALS.md`.
  - Запрещено хранение паролей в Git.

## 5. Workflow Standards
- **Plan -> Act -> Validate:**
  1. Детальное планирование (в `Plan Mode` или через согласование плана).
  2. Исполнение последовательности шагов без прерываний после утверждения плана.
  3. Валидация изменений (тесты, проверка метаданных, актуализация документации).
- **Automation:** Всегда используйте навыки, описанные в `.gemini/skills/automation-guide/SKILL.md`.
