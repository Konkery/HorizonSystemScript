## HORIZON SERVER SYSTEM SCRIPT

Репозиторий содержит ПО и скрипты, предназначенные для автоматизированного развертывания платформы автоматизации `Horizon Automation` на устройствах `HUB` (Raspberry Pi CM4).

### Основная концепция: Динамическая инфраструктура

Система Horizon Automation переведена на модель "Инфраструктура как Код" (IaC). Все компоненты системы формируются динамически на основе параметров:
*   **Файл настроек**: `res/01__config/settings.json`
*   **Масштабируемость**: Изменение количества инстансов (от 1 до 5) выполняется сменой одного ключа `instancesCount`.
*   **Уникальность**: Имена баз данных включают префикс вендингового аппарата (`machineName`), что позволяет централизованно собирать данные с хабов без конфликтов.
*   **Безопасность**: Пароли генерируются случайно при развертывании и сохраняются в `res/01__config/generated_secrets.json`.

---

### Системные требования

Для корректной работы скриптов на устройстве HUB должны быть установлены:
*   **Node.js** (версия 16.x или выше)
*   **MongoDB** (версия 6.x или выше)
*   **systemd** (стандарт для Linux дистрибутивов)

---

### Быстрый старт (Порядок развертывания)

Для полной инициализации системы на новом устройстве выполните следующие шаги:

0.  **Установка зависимостей** (выполняется один раз в корне проекта):
    ```bash
    npm install
    ```

1.  **Генерация инфраструктуры**:
    ```bash
    cd js/app/LinuxScript
    node generate-services-rev.03-v01.js
    ```
    *Результат: Созданы пароли в `res/01__config/CREDENTIALS.md` и службы в `NodeRed/`.*

2.  **Настройка MongoDB**:
    ```bash
    cd ../MongoDBscript
    node 01__MongoDB-CreateUserAdmin-rev.03-v01.js   # Создать root
    node 02__MongoDB-InitParam-rev.03-v01.js         # Настроить доступ (требует sudo)
    node 03__MongoDB-CreateUserAndCollection-rev.03-v01.js # Создать БД и коллекции
    ```

### Директория `js\app`

#### 1. LinuxScript (Управление ОС и службами)
*   `generate-services-rev.03-v01.js` — **Точка входа.** Генерирует системные службы, сбрасывает пароли и очищает рабочие директории.
*   `pb-rev.03-v01.sh` — Настройка безопасного выключения системы.
*   `README-LinuxScript.md` — Подробное руководство по автоматизации ОС.

#### 2. MongoDBscript (Управление данными)
*   `01__MongoDB-CreateUserAdmin-rev.03-v01.js` — Создание суперпользователя.
*   `02__MongoDB-InitParam-rev.03-v01.js` — Системная настройка СУБД.
*   `03__MongoDB-CreateUserAndCollection-rev.03-v01.js` — Инициализация прикладной структуры: создание динамических БД, пользователей и коллекций (`TimeSeries`, `Capped`).
*   `README-MongoDBscript.md` — Описание реестра БД и типов коллекций.

#### 3. NodeRedScript (Среда исполнения)
*Инструменты данной директории находятся в процессе миграции на новую систему конфигурации.*

#### 4. Common/ (Общие инструменты)
* **Common/** — Библиотека общих инструментов. Содержит модуль ConfigLoader для централизованной настройки и безопасности.

### Логика формирования ресурсов

| Ресурс | Правило формирования | Пример |
| :--- | :--- | :--- |
| **Пользователь** | `operator` + `[ID]` | `operator1` |
| **База данных** | `[MachineName]` + `_dbHA` + `[ID]` | `MAS-n1701_dbHA1` |
| **Порт Node-RED** | `startPort` + `([ID] - 1)` | `2001`, `2002`... |
| **Пароль** | Случайная строка (8 симв.) | `k9#F2sL! ` |
