# README-Deploy: Автоматизация развертывания на Хабы

Этот модуль предназначен для дистанционного обновления ПО на промышленных устройствах Horizon Automation Hub (Raspberry Pi CM4/CM5). Скрипт автоматизирует полный цикл развертывания через SSH-соединение с использованием архивного метода передачи данных.

---

## Назначение и возможности

Модуль `deploy-hub.js` выполняет следующие операции:

### Основные функции:
1. **Автоматическое чтение SSH-конфигураций** из `~/.ssh/config` (все хосты с префиксом `hub*`)
2. **Создание оптимизированного tar.gz архива** проекта (только необходимые файлы)
3. **Параллельная загрузка** на несколько хабов
4. **Автоматическая распаковка** и развертывание на целевых системах
5. **Детальное логирование** всех операций с формированием итоговой сводки

### Процесс развертывания на каждый хаб:
1. Установка SSH-соединения с аутентификацией по приватному ключу
2. Открытие SFTP-сессии
3. Очистка старых файлов (`~/DEPLOY`, старые архивы)
4. Загрузка нового архива через SFTP
5. Распаковка архива в `~/DEPLOY`
6. Удаление архива с хаба (экономия места)
7. Закрытие соединения

---

## Архитектура и оптимизация

### Создание архива
Модуль создает **оптимизированный** tar.gz архив, включающий только необходимые элементы:
- `js/` - исполняемые модули проекта
- `res/` - ресурсы и конфигурации
- `sh/` - shell-скрипты
- `package.json`, `package-lock.json` - зависимости Node.js
- `README.md` - документация
- `.gitignore`, `.prettierrc` - конфигурационные файлы

**Исключаются**: `node_modules`, временные файлы, `.git`, кэши, логи.

### Парсер SSH-конфигураций
Функция `getHubConfigs()` автоматически:
- Читает `~/.ssh/config` построчно
- Идентифицирует блоки Host с префиксом "hub"
- Извлекает параметры: `HostName`, `User`, `IdentityFile`
- Преобразует относительные пути ключей (~/) в абсолютные
- Загружает содержимое приватных ключей в память
- Возвращает массив готовых конфигураций для ssh2.Client

**Формат SSH config:**
```
Host hub-*
    HostName <ip-address>
    User <dynamic-user>
    IdentityFile ~/.ssh/id_rsa_hub
```

## Конфигурация

### Настройки в deploy-hub.js:
- `SSH_CONFIG_PATH` - путь к SSH конфигу (по умолчанию `~/.ssh/config`)
- `SETTINGS_PATH` - путь к `settings.json` (автоматически определяется)
- `CONNECT_TIMEOUT` - таймаут операций (10 сек)
- `DEBUG` - режим отладки (true/false)
- `ARCHIVE_NAME` - имя архива (`horizon-system-script.tar.gz`)

### Параметры из settings.json:
- `system.hubIp` - IP-адрес хаба (для логирования)
- `system.vending.instancesCount` - количество обрабатываемых хабов

## Требования

### Системные требования:
- **Node.js** с модулем `ssh2` (устанавливается через `npm install`)
- **Git for Windows** (для tar-архивации в Windows)
- Настроенные **SSH-ключи** для доступа к хабам
- Корректный файл `~/.ssh/config` с алиасами `hub*`

### Права доступа:
- Права на запись в корневой директории проекта (для создания архива)
- SSH-доступ к целевым хабам под указанными учетными записями
- Права на создание директорий в домашнем каталоге хаба (`~/DEPLOY`)

## Запуск

### Из корня проекта:
```bash
node js/app/LinuxScript/Deployment/deploy-hub.js
```

### Процесс выполнения:
1. Загрузка настроек из `settings.json`
2. Парсинг SSH-конфигураций из `~/.ssh/config`
3. Создание tar.gz архива проекта (~ несколько МБ)
4. Последовательное развертывание на каждый хаб из списка
5. Удаление локального архива после завершения
6. Вывод итоговой сводки: успешные/провальные развертывания

## Обработка ошибок

### Критические ошибки (прерывают выполнение):
- Невозможность загрузить `settings.json`
- Отсутствие SSH-конфигураций для хабов
- Невозможность создать архив проекта
- Отсутствие Git Bash (tar недоступен)

### Некритические ошибки (логируются, процесс продолжается):
- Ошибка подключения к отдельному хабу
- Таймаут операции на хабе (CONNECT_TIMEOUT = 10 сек)
- Ошибка распаковки на хабе
- Невозможность удалить локальный архив после завершения

### Таймауты:
- **Общая операция на хаб**: 10 секунд (CONNECT_TIMEOUT)
- **SSH ready timeout**: 20 секунд
- **Создание архива**: 30 секунд

## Примечания

- Скрипт работает в **последовательном режиме** (не параллельном) для стабильности
- Ошибка на одном хабе **не прерывает** обработку остальных
- Количество обрабатываемых хабов = `min(найденных SSH-конфигураций, instancesCount)`
- Локальный архив автоматически удаляется после завершения всех операций
- При DEBUG=true выводится содержимое распакованной директории `~/DEPLOY`
