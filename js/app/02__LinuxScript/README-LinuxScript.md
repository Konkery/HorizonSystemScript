# LinuxScript: Системные службы и автоматизация ОС

Данная директория содержит инструменты для настройки операционной системы Linux и управления жизненным циклом служб Node-RED на устройствах Horizon Hub.

**ВНИМАНИЕ:** Все команды в данном руководстве должны запускаться исключительно из **корневой директории проекта** (`HorizonSystemScript/`).

---

## Структура раздела

```
LinuxScript/
├── generate-services.js        # Генератор systemd-служб
├── NodeRedServices/            # Иерархия сгенерированных служб
│   ├── hubLOW/
│   ├── hubMID/
│   └── hubHI/
└── Deployment/                 # Модуль удаленного развертывания
    ├── deploy-hub.js
    └── README-deploy.md
```

---

## Модули

### 1. Deployment (Развертывание)

**Модуль:** `Deployment/deploy-hub.js`

**Назначение:** Автоматизированное дистанционное развертывание проекта Horizon Automation на промышленные хабы через SSH.

**Подробнее:** [README-deploy.md](Deployment/README-deploy.md)

**Запуск:**

```bash
node js/app/LinuxScript/Deployment/deploy-hub.js
```

---

### 2. Генератор служб (`generate-services.js`)

Этот модуль динамически создает конфигурационные файлы `systemd` (`.service`) для инстансов Node-RED.

- **Функционал:**
  - **Иерархия**: Генерирует службы с распределением по директориям хабов (`hubLOW`, `hubMID`, `hubHI`).
  - **Динамическая конфигурация**: Читает настройки из `res/01__config/settings_ha.json`, используя параметры `countUsers` и `userTemplate` для каждого хаба.
  - **Изоляция**: Полностью очищает поддиректории хабов перед каждой генерацией.
- **Запуск (из корня проекта):**
  ```bash
  node js/app/LinuxScript/generate-services.js
  ```
  _Скрипт использует глобальную конфигурацию для всех хабов, но разделяет файлы юнитов по соответствующим папкам._
