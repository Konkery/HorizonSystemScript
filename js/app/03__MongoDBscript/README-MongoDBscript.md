# MongoDBscript: Динамическое управление СУБД Horizon Automation

Директория содержит инструменты для автоматизированного развертывания и управления СУБД MongoDB в трехуровневой архитектуре. Основная особенность — полная динамическая зависимость от конфигурационного файла `res/01__config/settings_ha.json`.

## Реестр баз данных и динамическое именование

Система Horizon Automation использует иерархическую структуру имен для обеспечения уникальности данных.

### 1. Формирование имени БД

Имя каждой базы данных формируется динамически при старте скриптов:
`[Имя_Аппарата][ПОСТФИКС_БД][Номер_Инстанса]`

Параметр `dbPostfix` задается в конфигурации для каждого хаба отдельно.

---

## Порядок выполнения скриптов (из корня проекта)

**ВНИМАНИЕ:** Все команды должны запускаться исключительно из **корневой директории проекта** (`HorizonSystemScript/`).

Для работы скриптов **обязательно** указывать целевой хаб через аргумент `--hub`.

1.  **`node js/app/03__MongoDBscript/01__MongoDB-CreateUserAdmin.js --hub=[hubLOW|hubMID|hubHI]`**
    - Настройка суперпользователя `admin` (root) для указанного хаба.
2.  **`sudo node js/app/03__MongoDBscript/02__MongoDB-InitParam.js --hub=[hubLOW|hubMID|hubHI]`**
    - Настройка безопасности MongoDB (authorization, bindIp). **Требует sudo**.
3.  **`node js/app/03__MongoDBscript/03__MongoDB-CreateUserAndCollection.js --hub=[hubLOW|hubMID|hubHI]`**
    - Динамическая инициализация: создание БД, пользователей и коллекций согласно шаблонам хаба.
