# NodeRedScript: Локальная конфигурация инстансов Node-RED

Директория содержит модули для окончательной настройки среды исполнения Node-RED непосредственно на устройстве Horizon Hub.

## Принцип работы

Настройка выполняется инженером последовательно для каждого инстанса. Необходимо зайти на Хаб по SSH под соответствующим пользователем (`operator1`, `operator2` и т.д.).

**ВНИМАНИЕ:** Все команды в данном руководстве должны запускаться исключительно из **корневой директории проекта** (`HorizonSystemScript/`).

---

## Порядок выполнения модулей

Часть модулей должны запускаться с привилегиями `sudo`, так как они взаимодействуют с системными службами.

### 1. `01__NodeRed-InitParam.js`

Выполняет базовую настройку сетевого интерфейса и системных параметров.

- **Действие**: Разрешает удаленные подключения и устанавливает TCP-порт на основе конфигурации хаба.
- **Запуск (из корня проекта)**:
  ```bash
  node js/app/04__NodeRedScript/01__NodeRed-InitParam.js --hub=[hubLOW|hubMID|hubHI]
  ```

### 2. `02__NodeRed-CreateUsers.js`

Настраивает систему безопасности и права доступа.

- **Изотропная модель**: В каждом инстансе регистрируются операторы, заданные в конфигурации.
- **Безопасность**: Использует пароли из `generated_secrets.json`, генерирует хеши для Node-RED.
- **Запуск (из корня проекта)**:
  ```bash
  sudo node js/app/04__NodeRedScript/02__NodeRed-CreateUsers.js --hub=[hubLOW|hubMID|hubHI]
  ```
  _После выполнения данного скрипта системная служба Node-RED будет автоматически перезапущена._

---

## Техническое примечание

Актуальные пароли для хеширования скрипты берут из `res/01__config/generated_secrets.json`. Перед началом работы убедитесь, что конфигурация `settings_ha.json` актуальна.
