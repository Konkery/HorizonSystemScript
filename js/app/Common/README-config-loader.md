# README-ConfigLoader: Библиотека конфигурации и безопасности

Модуль `config-loader-rev.03-v01.js` является ядром системы автоматизации Horizon Automation. Он обеспечивает логическую связь между статическими настройками пользователя и динамическими требованиями инфраструктуры.

## Назначение
Модуль решает три ключевые задачи:
1.  **Динамическое расширение**: Превращает базовые параметры из `settings.json` в полный список объектов (инстансов) с вычисленными именами БД и портами.
2.  **Управление безопасностью**: Автоматически генерирует криптостойкие пароли для всех пользователей системы.
3.  **Единый источник истины**: Предоставляет всем остальным JS-модулям унифицированный интерфейс для получения конфигурации.

---

## Основные функции

### `loadConfig()`
Главная точка входа. Выполняет следующие действия:
*   Валидирует наличие файла `res/01__config/settings.json`.
*   Вызывает генератор секретов.
*   Формирует массив `instances` на основе `vending.instancesCount`.
*   Возвращает объект конфигурации, готовый к использованию в MongoDB или Node-RED скриптах.

### `clearSecrets()`
Функция управления жизненным циклом паролей:
*   Удаляет файл `res/generated_secrets.json`.
*   Удаляет текстовый отчет `res/CREDENTIALS.md`.
*   Обычно вызывается в начале цикла развертывания (в `generate-services.js`).

### `generatePassword(length)`
Внутренняя утилита для создания случайных паролей (8 символов по умолчанию). Использует набор символов: буквы (A-z), цифры (0-9) и спецсимволы (!@#$%^&*).

---

## Генерируемые ресурсы

Модуль автоматически создает и обновляет следующие файлы в директории `res/01__config/`:

1.  **`generated_secrets.json`**: Технический файл в формате JSON. Используется для сохранения паролей между запусками различных модулей.
2.  **`CREDENTIALS.md`**: Человекочитаемый отчет в виде таблиц. Содержит логины, пароли, порты и имена БД для всех операторов и сервисов.

---

## Техническое взаимодействие

Для использования модуля в других скриптах используется стандартный механизм `require`:

```javascript
const { loadConfig, clearSecrets } = require('../Common/config-loader-rev.03-v01');

// Получение конфигурации
const config = loadConfig();
console.log(config.instances[0].dbName); // Пример: MAS-n1701_dbHA1
```

## Правила ревизии
Любое изменение в алгоритме формирования имен БД или логике распределения портов требует инкремента **ревизии** модуля, так как это нарушает обратную совместимость с ранее развернутыми Хабами.
