# LinuxScript: Системные службы и скрипты управления

Данная директория содержит конфигурационные файлы системных служб `systemd` и вспомогательные Bash-скрипты для настройки операционной системы Linux на устройствах Horizon Hub.

## Состав директории

### 1. Службы Node-RED (`nodered1.service` - `nodered5.service`)

Эти файлы предназначены для настройки многопользовательской среды исполнения Node-RED. Каждый сервис запускает отдельный экземпляр Node-RED для конкретного оператора.

*   **Назначение:** Обеспечение изолированной работы до 5 независимых потоков автоматизации.
*   **Параметры запуска:**
    *   **User/Group:** Соответствующий пользователь (`operator1`...`operator5`).
    *   **Порты:** Каждый экземпляр настроен на свой порт (по умолчанию 2001-2005, настраивается в `settings.js`).
    *   **Автозапуск:** Службы настроены на автоматический перезапуск при сбое (`Restart=on-failure`).
*   **Использование:**
    1. Скопировать файлы в `/etc/systemd/system/`.
    2. Обновить конфигурацию: `sudo systemctl daemon-reload`.
    3. Включить автозапуск: `sudo systemctl enable nodered1`.
    4. Запустить службу: `sudo systemctl start nodered1`.

### 2. Скрипт безопасного выключения (`pb.sh`)

Скрипт предназначен для настройки корректной обработки нажатия физической кнопки питания на Raspberry Pi.

*   **Функционал:**
    *   Перехватывает нажатие кнопки питания (`HandlePowerKey=ignore` в `logind.conf`).
    *   Настраивает демон `triggerhappy` для вызова скрипта выключения.
    *   Создает исполняемый файл `/usr/local/bin/blitz.shutdown.sh`, который выполняет команду `shutdown --poweroff now`.
    *   Настраивает права `sudoers` для возможности выполнения выключения без ввода пароля.
*   **Запуск:**
    ```bash
    sudo chmod +x pb.sh
    sudo ./pb.sh
    ```

## Инструкция по развертыванию

1. **Подготовка пользователей:** Убедитесь, что в системе созданы пользователи `operator1` - `operator5`.
2. **Настройка сервисов:** Скопируйте нужные `.service` файлы в системную директорию и активируйте их.
3. **Безопасное питание:** Запустите `pb.sh` для настройки аппаратной кнопки выключения. Это критически важно для предотвращения повреждения файловой системы при некорректном отключении питания.
