# LinuxScript: Системные службы и автоматизация ОС

Данная директория содержит инструменты для настройки операционной системы Linux и управления жизненным циклом служб Node-RED на устройствах Horizon Hub.

## Новое: Полная динамическая генерация

В текущей версии реализован переход к полностью динамической модели развертывания. Параметры инстансов больше не прописываются вручную, а вычисляются на лету.

### 1. Генератор служб (`generate-services-rev.03-v01.js`)

Этот модуль является "точкой входа" в процесс развертывания. 

*   **Функционал:**
    *   **Сброс безопасности:** При каждом запуске вызывает `clearSecrets()`, что приводит к генерации **новых случайных паролей** для всех пользователей системы (операторов и сервисов).
    *   **Глубокая очистка:** Удаляет ВСЕ файлы в директории `js/app/LinuxScript/NodeRed/`, которые не являются `.js` исходниками. Это гарантирует отсутствие "мусора" от предыдущих конфигураций.
    *   **Динамическое распределение:** Создает `.service` файлы на основе `instancesCount` из `settings.json`.
*   **Динамические параметры:**
    *   **Порты:** Назначаются начиная со значения `startPort` (по умолчанию 2001) и увеличиваются на 1 для каждого следующего инстанса.
    *   **Пользователи:** Формируются по шаблону `operatorN`.
*   **Запуск:**
    ```bash
    node generate-services-rev.03-v01.js
    ```

## Реестр портов (логика формирования)

Теперь распределение портов для служб Node-RED не является статическим. Если в `settings.json` указано `startPort: 2001`, то:
*   **Инстанс №1:** порт 2001
*   **Инстанс №2:** порт 2002
*   ... и так далее до значения `instancesCount`.

Эти значения автоматически подставляются генератором в параметр `--port` при формировании systemd-юнитов.
