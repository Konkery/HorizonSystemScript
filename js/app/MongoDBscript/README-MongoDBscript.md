# MongoDBscript: Инициализация и настройка базы данных

Данная директория содержит набор JavaScript-модулей (для Node.js), предназначенных для автоматизированного развертывания и настройки СУБД MongoDB в составе платформы Horizon Automation.

## Состав и порядок выполнения скриптов

Процесс настройки разделен на три последовательных этапа. Скрипты необходимо запускать строго в указанном порядке.

### 1. Создание системного администратора (`01__MongoDB-CreateUserAdmin-rev.01-v01.js`)

На чистой установке MongoDB без активной аутентификации этот скрипт создает первого пользователя с правами суперпользователя.

*   **Назначение:** Создание пользователя `admin` с ролью `root` в базе данных `admin`.
*   **Действие:** Позволяет в дальнейшем управлять СУБД с включенным механизмом авторизации.
*   **Запуск:** `node 01__MongoDB-CreateUserAdmin-rev.01-v01.js`

### 2. Инициализация системных параметров (`02__MongoDB-InitParam-rev.02-v04.js`)

Скрипт выполняет низкоуровневую настройку конфигурации MongoDB на уровне операционной системы.

*   **Назначение:** Подготовка СУБД к промышленной эксплуатации.
*   **Выполняемые действия:**
    1. Создание резервной копии `/etc/mongod.conf`.
    2. Разрешение удаленных подключений (`bindIp: 0.0.0.0`).
    3. Включение механизма аутентификации (`authorization: enabled`).
    4. Перезапуск службы `mongod`.
*   **Запуск:** `sudo node 02__MongoDB-InitParam-rev.02-v04.js` (требуются права `sudo`).

### 3. Создание прикладной структуры БД (`03__MongoDB-CreateUserAndCollection-rev.01-v05.js`)

Основной скрипт для формирования рабочего окружения автоматизации.

*   **Назначение:** Полная инициализация баз данных, коллекций и прав доступа для операторов и системных служб.
*   **Структура данных:**
    *   **Базы данных:** `dbFramework1` - `dbFramework5` (для операторов), `dbGraylog`, `dbSystem`.
    *   **Типы коллекций:**
        *   `rawData`, `workData`: **TimeSeries** (для эффективного хранения временных рядов с автоудалением через 10 суток).
        *   `archiveData`, `loggingData`: **Capped** (кольцевой буфер объемом 50 МБ).
        *   Остальные: Стандартные коллекции.
    *   **Пользователи:** Создаются пользователи `operator1`-`operator5`, `graylog`, `system` с ролью `dbOwner` для соответствующих баз данных.
*   **Запуск:** `node 03__MongoDB-CreateUserAndCollection-rev.01-v05.js`

## Технические детали

*   **Среда исполнения:** Node.js.
*   **Зависимости:** Пакет `mongodb` (устанавливается через `npm install`).
*   **Безопасность:** Все пароли и учетные данные жестко заданы в коде скриптов в соответствии с проектной документацией. При изменении паролей необходимо синхронизировать их со скриптами в директории `NodeRedScript`.

## Важные примечания

Скрипт №3 является деструктивным: при запуске он **удаляет** существующие коллекции и пользователей в целевых базах данных перед их повторным созданием. Используйте его только для первичной настройки или полного сброса окружения.
