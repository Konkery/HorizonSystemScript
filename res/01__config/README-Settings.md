# README-Settings: Документация файла конфигурации

Файл `settings.json` является центральным узлом управления всей инфраструктурой Horizon Automation. Он определяет параметры масштабирования, сетевые настройки и структуру хранения данных.

## Местоположение
*   **Путь к файлу:** `res/01__config/settings.json`
*   **Управление:** Редактируется вручную инженером перед началом развертывания.

---

## Структура файла

### 1. Секция `system` (Общие настройки)
Определяет параметры подключения к СУБД и характеристики конкретного вендингового аппарата.

*   **`mongoUri`**: Строка подключения к серверу MongoDB.
*   **`admin`**: Учетные данные суперпользователя (root) для управления СУБД.
*   **`vending`**: Ключевой блок для динамической генерации:
    *   `machineName`: Полный серийный номер аппарата. Используется как префикс для имен БД.
    *   `instancesCount`: Количество инстансов (1-5). Определяет число создаваемых БД, пользователей и служб.
    *   `startPort`: Начальный порт для Node-RED (например, 2001).
    *   `dbBaseName`: Базовая часть имени БД (например, `dbHA`).
    *   `userBaseName`: Базовая часть логина оператора (например, `operator`).

### 2. Секция `collectionTemplates` (Шаблоны данных)
Описывает структуру коллекций, которые будут автоматически созданы в каждой базе данных инстанса.

*   **TimeSeries**: Коллекции для высоконагруженных данных с датчиков.
    *   `timeField`: Поле метки времени.
    *   `metaField`: Поле метаданных (ID датчика).
    *   `expireAfterSeconds`: TTL (срок хранения данных).
*   **Capped**: Кольцевые коллекции фиксированного объема.
    *   `size`: Максимальный размер в байтах.
*   **Standart**: Обычные коллекции без ограничений по объему и времени.

### 3. Секция `externalServices` (Внешние сервисы)
Статическая конфигурация для сторонних компонентов (Graylog, System и т.д.). Для них также генерируются динамические пароли, но имена БД и пользователей фиксированы.

---

## Правила редактирования

1.  **Точки в именах**: Избегайте использования точек в ключе `machineName`. MongoDB не рекомендует использовать точки в названиях БД. Заменяйте их на дефисы.
2.  **Формат**: Файл должен соответствовать стандарту JSON. Для описания полей используйте ключ `description` или `_comment`, так как стандартный JSON не поддерживает комментарии.
3.  **Безопасность**: Не вносите в этот файл пароли операторов. Они генерируются автоматически и сохраняются в `res/CREDENTIALS.md`.

---

## Техническое примечание
Валидация и чтение этого файла осуществляется модулем `js/app/Common/config-loader-rev.03-v01.js`. Любые изменения в структуре JSON должны быть синхронизированы с логикой этого модуля.
